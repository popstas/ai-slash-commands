# todo-review

Обзор статуса всех проектов через их `docs/TODO.md`: где что в работе, где застряло, где пусто.

## Что делать

1. **Найди файлы.** git-проекты в `~/projects`, у которых рядом с папкой `docs/` есть `.git`:
   ```bash
   find ~/projects -type f -path '*/docs/TODO.md' 2>/dev/null | while read f; do
     d=$(dirname "$(dirname "$f")")
     [ -e "$d/.git" ] && echo "$d"
   done | sort
   ```
   Корень проекта = родитель папки `docs/`. `.git` может быть директорией или файлом (worktree) — поэтому `-e`, а не `-d`.

2. **Выведи по каждому проекту** заголовок с путём и полное содержимое его `docs/TODO.md` (читай через Read). Пустой файл помечай «пусто».

3. **Сводка в конце** одним блоком: сколько проектов всего, у кого есть незакрытые пункты (`- [ ]`, `- [/]` или нечекнутые буллеты), где TODO пустой/закрыт, и 1–2 строки общего вывода (что горит, что заброшено). Только по факту из файлов — не выдумывай.

## Замечания

- Не редактируй `docs/TODO.md` — это рабочие файлы проектов.
- Если файлов много и они большие — в сводке приоритет незакрытым пунктам, но полный текст показывай всё равно.
- Запустить работу по конкретному TODO — это скилл `do` (`/do` в нужном проекте).
